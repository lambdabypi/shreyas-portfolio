📅 MARCH – APRIL 2026

⚙️ Behind the Scenes - Servers & Infrastructure

This is everything that happens on our servers to make the app work - users never see this directly, but it affects speed, reliability, and what the app can do.

☁️ Goals - Cloud Sync

Goals Saved to the Cloud

New Feature
  • The server now has dedicated storage for user goals - create, update, complete, and delete, all synced to the cloud
  • Goals survive if the user reinstalls the app or switches phones
  • The database was updated with a new schema to store goal and user data
💡 In plain terms: Your goals aren't just saved on your phone - they're backed up to our servers so they're never lost.

🔑 Password Reset - Simplified

6-Digit Reset Code

Improvement
  • Password reset now sends a simple 6-digit code (like a bank OTP) instead of a long random string
  • Much easier to type on a phone - matches the new reset dialog in the app

💳 Payments - Stripe Fix

Real Payment Methods

Bug Fix
  • A bug was found where the server was using a hardcoded test card for all payments - meaning real payment methods were being ignored
  • This is now fixed: the server correctly processes whichever payment method the user submits
💡 In plain terms: Payments will now actually use the card or method the user provides, not a dummy test card from development.

📈 Stock Trading Server (Alpaca)

Trading Engine Connected

New Integration
  • Our Python server now communicates directly with Alpaca's trading platform
  • Supports real-time updates via a live data connection (WebSocket)
  • Trades and payments are now properly linked to individual user accounts at the database level

🚀 Performance - Database Connection Pool

Faster Server Response Times

  • The server now maintains a pool of ready-to-use database connections (up to 20 at once) instead of opening a fresh connection for every request
  • This significantly reduces response time under load
💡 In plain terms: Instead of unlocking a new door every time someone knocks, we now keep 20 doors already open and ready - the app responds faster.

🛠️ Developer Experience

Easier Local Setup

DevOps
  • A convenience script was added so developers can start the server with one command
  • Sensitive configuration (API keys, database passwords) moved out of the code and into a separate private file that never gets uploaded to GitHub

Startup Checks

Reliability
  • The server now checks that all required credentials (database, trading API, payments) are present before starting
  • If something is missing, it fails immediately with a clear error - rather than crashing unexpectedly mid-operation
💡 In plain terms: The server now does a pre-flight checklist before takeoff.
📅 MARCH – APRIL 2026

🔒 Security - Keeping Users Safe

A significant amount of work this sprint went into hardening the security of the app and its servers. Here's what changed and why it matters.

📦 Encrypted Session Storage

Login Sessions Stored Safely on Device

Security Upgrade
  • Previously, login session data was stored in a basic file on the device - readable by other apps in some scenarios
  • It's now stored in an encrypted vault using AES-256 encryption (the same standard used by banks)
  • Even if someone gained access to the device's storage, the session data would be unreadable
💡 In plain terms: Your login is now stored like a combination-locked safe, not a sticky note on your desk.

🔄 "Stay Logged In" Fixed

App Remembers You Correctly

Bug Fix
  • After closing and reopening the app, users were sometimes being sent back to the login screen even though they were already authenticated
  • This is now fixed - the app correctly checks your session and takes you straight to your dashboard if you're already logged in
  • If your bank account was already connected, you skip the setup screen too

🎫 Token Refresh System

Automatic Login Token Renewal

Security Upgrade
  • Login tokens (think of them as temporary badges that prove who you are) now expire for security - but the app silently renews them in the background
  • Users stay logged in seamlessly without being interrupted by re-authentication prompts
  • This works for both the main server and the trading server independently
💡 In plain terms: Like a visitor badge that auto-renews before it expires - you never get kicked out mid-session.

🌐 Network Security

Encrypted Connections Only

Security Upgrade
  • The app now blocks all unencrypted (plain HTTP) connections by default - every request to the internet must use HTTPS
  • An exception is made only for the local development emulator, which doesn't need encryption
💡 In plain terms: The app now refuses to send data over an unsecured connection - like insisting on a sealed envelope instead of a postcard.

🔑 Secrets Out of the Code

No More Hardcoded Keys

Security Upgrade
  • API keys, server addresses, and payment credentials were previously written directly in the app's code - a common but risky practice
  • These are now stored in a separate configuration file that is never uploaded to GitHub or shared publicly
💡 In plain terms: We moved all the "passwords" out of the visible code and into a private safe.

🧹 Private Data Removed from Logs

Logs No Longer Expose User Info

Privacy
  • System logs (used by developers to debug issues) were inadvertently recording personal user information
  • This has been removed - logs now only contain technical data needed for debugging, nothing personally identifiable

🔒 Security Summary

  • ✅ Login sessions encrypted on-device with AES-256 (bank-grade encryption)
  • ✅ "Stay logged in" bug fixed - no more unexpected logouts
  • ✅ Auto-renewing login tokens across app and trading server
  • ✅ All network traffic forced over HTTPS
  • ✅ API keys and credentials moved out of the codebase
  • ✅ Personal data removed from system logs
  • ✅ Server endpoint access rules tightened across all routes
