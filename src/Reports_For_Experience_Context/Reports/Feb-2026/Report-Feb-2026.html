Strategic Product Pivot

⚡ Major Direction Change: Sports Betting → Lifestyle Challenges

Splitz pivoted from sports/finance/politics betting to personal lifestyle challenges between friends. This change improves legal positioning (friendly wager posture) and differentiates the product in a crowded market.

  • New categories: Fitness, Food, Productivity, Skills, Social, Lifestyle
  • Stakes capped at $500 to maintain "friendly wager" legal classification
  • Betting restricted to accepted friends only — enforced at API and UI level
  • Updated seed data with lifestyle bets (e.g. "Run a 5K in under 30 min", "No coffee for 30 days")

Bug Fixes

API Rate Limiting (429 Errors)

Limit of 100 req/15 min was too low for multi-call screen loads → Raised/bypassed in dev mode

Duplicate Posts in Feed

Race condition between useFocusEffect and socket events both fetching posts → Added deduplication in Redux addPost reducer

Comment 404 Error

Route used /:postId/comments but controller read req.params.id → Standardized all routes to /:id

Zero Reaction / Comment Counts

Missing post.increment() calls in controllers → Added increments + one-time SQL backfill for existing records

Username Display Inconsistency

Username showed differently between feed cards and detail screen → Unified data source

Duplicate Comments on Reconnect

Socket reconnect during profile updates caused double renders → Fixed reconnection handling
