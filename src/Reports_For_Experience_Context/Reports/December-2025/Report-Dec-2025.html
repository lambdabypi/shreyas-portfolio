🏦 Banking Intelligence API

Project Online

Major Security Incident & Recovery

Security Breach Discovery

December 7-11, 2025

  • Sophisticated cyberattack discovered on EC2 instance
  • Attackers gained root access through exposed SSH port (0.0.0.0/0)
  • Multiple persistent backdoors installed
  • Evidence of brute force attacks from external IPs

Crisis Response Timeline

Immediate Actions
  • Disabled malicious systemd services (lived.service, alive.service)
  • Removed backdoor scripts from /tmp/runnv/
  • Reset database master credentials
  • Secured Aurora RDS with VPC security groups
Infrastructure Rebuild
  • Complete migration to clean EC2 instance
  • Migrated from compromised instance (18.217.57.155) to clean (3.131.131.48)
  • Updated RDS security groups for new instance access
  • Reassociated Elastic IP
Security Hardening
  • Restricted SSH to specific IPs only
  • Implemented fail2ban for automated attack blocking
  • Configured firewalld for traffic filtering
  • Set up comprehensive logging and monitoring

Major Issues Resolved

Performance Optimization

Problem: 5-7 second startup pause

Cause: 18MB background video blocking render

Solution: Optimized nginx video serving with proper MIME types and caching

API Rate Limiting

Problem: Hitting Gemini API free tier limit

Impact: Banking reports failing to generate

Solution: Sequential processing with FREE_TIER_MODE configuration

Database Query Issues

Problem: Command reports not using actual transaction data

Cause: Date range issues in queries

Solution: Expand timeframe to 90 days

🎲 Splitz - Social Betting Mobile App

MVP Ready

Project Overview

What is Splitz?

A React Native mobile betting app that enables friends to create and participate in casual bets with each other. Fundamentally different from corporate prediction markets - designed for social, friend-to-friend betting rather than financial trading.

Tech Stack

Frontend

React Native Expo
  • Redux Toolkit with redux-persist
  • Socket.IO client for real-time updates
  • TypeScript for type safety
  • Custom animations using Animated API

Backend

Express.js PostgreSQL
  • Sequelize ORM
  • Socket.IO for WebSockets
  • JWT authentication
  • Cloudinary for image storage

Key Features

1. Authentication System

JWT
  • User registration with firstName/lastName
  • Persistent authentication via AsyncStorage
  • Profile management

2. Bet Management

Backend Frontend
  • Create bets with title, description, stakes
  • Privacy settings (public/private)
  • Join and resolve bets
  • Real-time updates

3. Social Proof Posts

Frontend
  • Share photos of betting moments
  • Reactions and comments
  • Cloudinary image uploads
  • Activity feed

4. Real-Time Features

Socket.IO
  • New bet notifications
  • Bet updates (joins, status changes)
  • Resolution notifications
  • Room-based architecture

5. Enhanced Animations

React Native
  • Dramatic dice drop on first login
  • Floating background elements
  • Celebration particle effects
  • Optimized typing detection

Major Bug Fixes (9 Resolved)

Currency Formatting

Stakes displayed as strings → Implemented formatCurrency() helper

Data Structure Mismatches

displayName vs firstName/lastName → Updated interfaces to match backend

Image Upload Failures

Unreliable FormData uploads → Switched to base64 encoding

Socket Event Naming

Inconsistent naming causing missed events → Standardized to camelCase

Animation Performance

Registration screen lag → Implemented debouncing during typing

Platform Date Pickers

Single picker didn't work well → Platform-specific implementations
