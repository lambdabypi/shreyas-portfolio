Splitz Development Report - Past Month
🎯 Project Overview
Splitz is a React Native mobile betting app that enables friends to create and participate in casual bets with each other. It's fundamentally different from corporate prediction markets - designed for social, friend-to-friend betting rather than financial trading.

🏗️ Architecture & Tech Stack
Frontend (React Native + Expo)

React Native with Expo for cross-platform development
Redux Toolkit with redux-persist for state management
Socket.IO client for real-time updates
TypeScript for type safety
Custom animations using Animated API

Backend (Node.js + Express)

Express.js REST API
PostgreSQL database with Sequelize ORM
Socket.IO for real-time WebSocket connections
JWT authentication
Cloudinary integration for image storage

Infrastructure

Local development server at IP 10.0.0.60:3000
Docker setup for PostgreSQL
Migrations and seeders for database management

🐛 Major Bug Fixes
1. Currency Formatting Issues

Problem: Bet stakes displayed as strings instead of formatted currency
Solution: Implemented proper formatCurrency() helper and ensured consistent usage across BetCard and BetDetails components

2. Data Structure Mismatches

Problem: Frontend expected displayName but backend used firstName/lastName
Solution: Updated TypeScript interfaces and components to match backend schema

3. Network Image Upload Failures

Problem: FormData image uploads unreliable on mobile
Solution: Switched to base64 encoding for reliable image transmission

4. Socket Event Naming

Problem: Inconsistent camelCase/PascalCase event names causing missed events
Solution: Standardized all socket events to camelCase (e.g., betUpdated, newBet)

5. Duplicate Bet Rendering

Problem: Bets appearing multiple times in HomeScreen
Solution: Implemented deduplication logic before rendering

6. Creator Restriction on Joining

Problem: Creators couldn't join their own bets
Solution: Removed artificial restriction in backend controller

7. Animation Performance

Problem: Registration screen lag during typing
Solution: Implemented typing detection with debouncing to pause heavy animations during input

8. Platform-Specific Date Picker

Problem: Single datetime picker implementation didn't work well across platforms
Solution: Implemented platform-specific pickers (combined for iOS, separate date/time for Android)

9. Icon Name Error

Problem: Used invalid Ionicons name "crown"
Solution: Changed to valid "star" icon
